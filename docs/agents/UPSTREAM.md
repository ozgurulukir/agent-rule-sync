## Checking for Upstream Changes

```bash
bin/rulepack bump                     # Check all git-sourced packages
bin/rulepack bump vibe-security       # Check single package
bin/rulepack bump --apply             # Auto-update pkgver + rebuild
bin/rulepack bump --apply vibe-security  # Update single package
```

`bump` compares the cached commit hash (from `build/index.yaml`) against the remote HEAD using `git ls-remote`. Status codes:

- `[CURRENT]` — cached matches remote HEAD
- `[CHANGED]` — upstream has new commits (pkgver update needed)
- `[UNKNOWN]` — no build index found (run `build` first)
- `[ERROR]` — network or access failure

With `--apply`, changed packages are:
1. Version bumped via `pkgver_func` (if defined) or date-based (`YYYY.MM.DD`)
2. PKGBUILD rewritten with new `pkgver` and `pkgrel: 1`
3. Cache invalidated for stale commit
4. Full rebuild triggered

## Adding an Upstream Source

To fetch from an upstream repository:

1. **Git source** (recommended for tracked upstream):
```yaml
source:
  - type: git
    url: https://github.com/example/repo.git
    ref: main
    path: path/to/file.md
    depth: 1
pkgver_func: "git log -1 --format=%cd --date=short 2>/dev/null | tr -d '-' || date +%Y.%m.%d"
```

2. **URL source** (single file, pinned by checksum):
```yaml
source:
  - type: url
    url: https://github.com/example/repo/raw/main/file.md
    sha256: "<checksum-of-content>"
```

3. **Local-path source** (symlink or copy from another local repo):
```yaml
source:
  - type: local
    path: /path/to/upstream/file.md
```

**Fetching**: `bin/rulepack build` automatically fetches URL/git sources (with SHA256/commit verification). Local sources are read directly.

## `pkgver_func` — Automatic Version Derivation

`pkgver_func` is the makepkg `pkgver()` parallel. When defined, `bin/rulepack bump --apply` and `bin/rulepack build` execute this shell command inside the fetched source directory to derive the package version automatically.

Common patterns:

| Pattern | Command | Output example |
|---------|---------|---------------|
| Date from last commit | `git log -1 --format=%cd --date=short \| tr -d '-'` | `20260526` |
| Latest tag | `git describe --tags --always 2>/dev/null` | `v1.2.3` |
| Fallback to date | `git describe --tags --always 2>/dev/null \|\| date +%Y.%m.%d` | `2026.05.26` |

## Upstream Skills

Custom skills are authored in `data/skills/`:

- `common/` — shared skill modules (e.g., `ast-grep.md`)
- `agent-specific/` — per-agent overrides (e.g., `goose-guardrails.md`, `droid-header.md`)

Vendor skills are auto-generated by `lib/rulepack/aggregate.rb` (do not edit).

## Upstream Maintenance (Manual)

To manually update an upstream URL source:
1. Fetch latest content manually or update URL
2. Compute new SHA256: `sha256sum <file>`
3. Update PKGBUILD `source[0].sha256`
4. Rebuild: `bin/rulepack build`

For local-path sources, update the file at the referenced path, then rebuild.
