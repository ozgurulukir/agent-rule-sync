# Upstream Sources

Tracking of upstream rule and skill sources for this SSoT installation.

## Rule Sources

| Rule | Source | Type | Path | Transformer |
|------|--------|------|------|-------------|
| memory | Original workstation | local | `ssot/packages/memory/src/` | copy |
| shell | Original workstation | local | `ssot/packages/shell/src/` | copy |
| vibe-security | raroque/vibe-security-skill | url | `https://github.com/raroque/vibe-security-skill/raw/main/SKILL.md` | copy |

**Note**: In the PKGBUILD system, upstream sources are declared per-package in the `source:` field. There is no central `schema.yaml` sources section.

## Adding an Upstream Source

To fetch from an upstream repository:

1. **URL source** in PKGBUILD:
```yaml
source:
  - type: url
    url: https://github.com/example/repo/raw/main/
    sha256: "<checksum-of-content>"
```

2. **Local-path source** (symlink or copy from another local repo):
```yaml
source:
  - type: local
    path: /path/to/upstream/file.md
```

**Fetching**: `ruby ssot/build.rb` automatically fetches URL sources (with redirect following and SHA256 verification). Local sources are read directly.

## Upstream Skills

Custom skills are authored in `ssot/skills/`:

- `common/` — shared skill modules (e.g., `ast-grep.md`)
- `agent-specific/` — per-agent overrides (e.g., `goose-guardrails.md`, `droid-header.md`)

Vendor skills are auto-generated by `aggregate-skills.rb` (do not edit).

## Upstream Maintenance

To update an upstream URL source:
1. Fetch latest content manually or update URL
2. Compute new SHA256: `sha256sum <file>`
3. Update PKGBUILD `source[0].sha256`
4. Rebuild: `ruby ssot/build.rb`

For local-path sources, update the file at the referenced path, then rebuild.
