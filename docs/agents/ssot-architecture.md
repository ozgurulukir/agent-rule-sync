## Concepts

### Sources

Upstream repositories and their platform origin. Defined in `schema.yaml` under `sources:`.

| Source ID | Platform | Type | Path / Base URL | Default Transformer |
|-----------|----------|------|-----------------|---------------------|
| `local` | generic | local | ‚Äî | `copy` |
| `tci` | opencode | local-path | `~/Projects/coderlm/agents/opencode` | `strip-frontmatter` |
| `vibe-security` | generic | url | `https://github.com/raroque/vibe-security-skill/raw/main/` | `copy` |

### Platforms

Agent platform formats and their transformers. Defined in `schema.yaml` under `platforms:`.

| Platform ID | Format | Transforms From | Custom Transformer |
|-------------|--------|-----------------|-------------------|
| `opencode` | directory | `[opencode, generic]` | ‚Äî (copy) |
| `claude` | import | `[opencode]` | `scripts/transforms/opencode-to-claude.rb` |
| `gemini` | import | `[opencode]` | `scripts/transforms/opencode-to-gemini.rb` |
| `qwen` | import | `[opencode]` | `scripts/transforms/opencode-to-qwen.rb` |
| `factory-droid` | skill | `[opencode]` | `scripts/transforms/opencode-to-droid.rb` |
| `goose` | skill | `[opencode]` | `scripts/transforms/opencode-to-goose.rb` |
| `crush` | skill | `[opencode]` | `scripts/transforms/opencode-to-crush.rb` |
| `oh-my-pi` | directory | `[opencode]` | `scripts/transforms/opencode-to-omp.rb` |

**Transform logic**: If `source.platform == agent.platform` ‚Üí no transform (copy). If different ‚Üí run custom transformer script (or built-in: `copy`, `strip-frontmatter`).

## Rules vs Skills

**Rules**: Workstation constraints (memory, shell, code-nav, patterns, git, security, TCI, vibe-security). Stored in `ssot/rules/`. Applied to all agents.
- Local rules (00‚Äì05): hand-written, versioned in repo.
- Upstream rules (06-tci, vibe-security): fetched via `make fetch-upstream`, transformed via `make transform`.

**Skills**: Agent-specific tips and guardrails.
- Custom skills: hand-written in `ssot/skills/` (e.g., `droid-header.md`, `goose-guardrails.md`, `ast-grep.md`).
- Upstream skills: declared in `schema.yaml` under `skills.upstream`, fetched and transformed like rules.
- Vendor skills: auto-generated by concatenating header + all rules + agent-specific skills + common skills + footer. Output to `ssot/skills/vendor/<agent>.md`.

## Agent Formats

| Format | Agents | Mechanism |
|--------|--------|-----------|
| `directory` | OpenCode, Claurst, Oh My Pi | Symlink each rule file from `ssot/rules/` to agent's rules directory |
| `import` | Gemini, Qwen | Write `@/abs/path` import lines into agent's config file |
| `skill` | Droid, Goose, Crush | Copy vendored skill file to agent's config location |

## Workflow

```bash
make setup          # fetch-upstream ‚Üí transform ‚Üí index ‚Üí vendor-skills ‚Üí sync (full install)
make fetch-upstream # Download upstream sources to ssot/vendor/ (local-path copy + URL fetch)
make transform      # Transform upstream ‚Üí SSoT (platform-aware)
make index          # Generate ssot/INDEX.md + ssot/index.json from schema
make vendor-skills  # Generate ssot/skills/vendor/ from rules+skills
make sync           # Distribute SSoT to all agent configs
make check          # Verify "All agents in sync"
make clean          # Remove generated/custom skill files (keeps vendor/)
```

## Adding a New Rule

1. Add entry to `ssot/schema.yaml` under `rules:` with `id`, `title`, `order`, `filename`, `source`.
2. If upstream: add `upstream_path` and optionally `transformer`.
3. If local: create `ssot/rules/<filename>` directly.
4. Run `make fetch-upstream` (if upstream) ‚Üí `make transform` ‚Üí `make index` ‚Üí `make vendor-skills` ‚Üí `make sync`.

## Adding a New Skill

1. Create `ssot/skills/<skill-id>.md` (custom) or declare upstream in `schema.yaml` under `skills.upstream`.
2. Reference in `schema.yaml` under `skills.agent-specific[agent]`.
3. Run `make vendor-skills && make sync`.

## Adding a New Upstream Source

1. Add source to `ssot/schema.yaml` under `sources:` with `id`, `platform`, `type` (`local-path` or `url`), `path` or `base_url`, `default_transformer`.
2. Add rules/docs/skills referencing that `source` with `upstream_path` and `filename`.
3. Run `make fetch-upstream` ‚Üí `make transform` ‚Üí `make index` ‚Üí `make vendor-skills` ‚Üí `make sync`.

## Adding a New Platform (Agent Type)

1. Add platform to `ssot/schema.yaml` under `platforms:` with `id`, `format`, `transforms.from` (list of source platforms), `transforms.custom` (transform script path, optional).
2. Add agent entry under `agents:` with `platform` reference and format-specific fields (`path`, `rules`, `skills`, `docs`, `header`).
3. If `transforms.custom` is set, write the transformer script (`scripts/transforms/<platform>.rb`) implementing a `Transform` class with a `#transform` method returning the transformed content.
4. Run `make sync`.

## Vendor Skill Generation

Each vendor skill file (`ssot/skills/vendor/<agent>.md`) contains:
- **Header**: inline from `agents[agent].header` or first agent-specific skill
- **All rules**: in order by `order` field (content from `ssot/rules/` after transform)
- **Agent-specific skills**: additional skills (excluding header skill if used)
- **Common skills**: shared skill modules (e.g., `ast-grep`)
- **Footer**: optional

Generated by `scripts/vendor-skills.rb`.

## Upstream Integration

See [docs/upstream-sources.md](../upstream-sources.md) for tracked upstream sources and fetch process.

Upstream sources are declared per-rule in `schema.yaml`. `fetch-upstream.rb` downloads raw content to `ssot/vendor/` (local-path copies files; URL fetches via HTTP with redirects). `transform.rb` then transforms upstream content into processed SSoT files (`ssot/rules/`, `ssot/docs/`, `ssot/skills/`) using:
- **Built-in transformers**: `copy` (identity), `strip-frontmatter` (remove YAML frontmatter)
- **Custom transformers**: platform-specific scripts (`scripts/transforms/*.rb`) or future LLM-based prompts

Transform log: `ssot/transforms.log` (YAML chunks, one per run).

## Index

`ssot/INDEX.md` (human-readable) and `ssot/index.json` (machine-readable) are auto-generated by `make index` from `schema.yaml`. They catalog all sources, platforms, rules, docs, skills, and agents.
