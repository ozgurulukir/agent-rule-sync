# SSoT Index Guide

## INDEX.md

`ssot/INDEX.md` is a human-readable catalog of the entire SSoT, auto-generated by `make index` from `schema.yaml`. Do not edit manually.

**Sections:**
- **Sources**: Upstream source definitions (id, platform, type, path/base_url, default_transformer)
- **Platforms**: Agent platform configurations (format, directories, transform rules)
- **Rules**: All workstation constraint rules (id, title, order, filename, source, upstream_path, transformer)
- **Docs**: Reference documentation (id, filename, source, upstream_path)
- **Skills**:
  - Common: shared skill modules (e.g., `ast-grep`)
  - Agent-specific: per-agent skill lists
  - Upstream: external skill definitions with source and transformer
- **Agents**: All configured agents (name, display_name, platform, format, path, rules, skills, docs)
- **Transform Log**: Last transform run summary (timestamp, count of transformed/skipped/errors)

## index.json

`ssot/index.json` is a machine-readable JSON version of the same data, generated alongside `INDEX.md`. Use for programmatic access (scripts, tooling).

**Structure:**
```json
{
  "version": "3.0",
  "generated": "2026-05-14T...",
  "sources": [...],
  "platforms": [...],
  "rules": [...],
  "docs": [...],
  "skills": {
    "common": [...],
    "agent_specific": { "opencode": [...], ... },
    "upstream": { "tci-analyze": {...}, ... }
  },
  "agents": [...]
}
```

## When to Regenerate

Run `make index` whenever:
- `schema.yaml` changes (add/remove sources, platforms, rules, docs, skills, agents)
- Rule metadata (title, order, filename) changes
- Upstream source configuration changes

`make setup` and `make transform` do NOT automatically regenerate the index. Run `make index` explicitly or add it to your workflow.

## Using the Index

- **Manual browsing**: Open `ssot/INDEX.md` in a viewer to see the full SSoT state.
- **Scripting**: Parse `ssot/index.json` for automated checks (e.g., verify all agents have required rules).
- **Audit**: Compare `INDEX.md` across commits to track changes to SSoT structure.

## Example: Verify Agent Rule Coverage

```bash
# Check that all agents have at least N rules
jq '.agents[] | .name, (.rules | length)' ssot/index.json
```
