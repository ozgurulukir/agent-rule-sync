## 2024-05-18 - [Tar Slip Vulnerability in HTTP Fallback Tarball Extraction]
**Vulnerability:** A path traversal vulnerability was present in `lib/rulepack/source.rb`'s `extract_tar_gz` method. It failed to restrict extracted paths to the destination directory due to unsafe reliance on string manipulation (`parts[1..-1].join('/')`) rather than verifying the resolved absolute path.
**Learning:** Even though the HTTP fallback extraction routine skips the top-level directory and splits the path using `/`, path traversal payloads with `..` can bypass these naive checks and create files outside the temporary git clone fallback directory.
**Prevention:** Always use `File.expand_path` to resolve the final destination path and verify that it strictly starts with the expanded destination directory path (`dest_path.start_with?(expanded_dest_dir + File::SEPARATOR)`) before creating directories or files.

## 2026-07-14 - [Symlink Traversal and Overwrite during Git Clone and Directory Iteration]
**Vulnerability:** A symlink traversal vulnerability existed during Git extraction in `cache.rb`, where `cleanpath` was used to validate git paths rather than `realpath`. Furthermore, `build_per_pkg.rb` copied directories containing symlinks using `cp_r(preserve: false)` and subsequently modified their contents with `File.write` via `Dir.glob`, leading to arbitrary file overwrite if a symlink pointed outside the build directory.
**Learning:** Checking a path with `cleanpath` does not resolve symlinks, which allows a malicious symlink in a git repo to bypass traversal checks and leak external files. Additionally, iterating through files with `Dir.glob` and blindly applying `File.write` is dangerous if the file happens to be a symlink, as Ruby's `File.write` follows the symlink and overwrites the target file.
**Prevention:** Always use `.realpath` instead of `.cleanpath` to resolve and validate file paths against their sandbox boundary. When iterating over files in a directory that may contain untrusted symlinks, always explicitly check and skip symlinks (`next if File.symlink?(path)`) before operating on them.

## 2026-07-17 - [Argument Injection in Git Clone Command]
**Vulnerability:** An argument injection vulnerability existed in `lib/rulepack/source.rb` where user-controlled `url` inputs were passed directly to `git clone` via `system(*cmd)` without an end-of-options separator (`--`).
**Learning:** Even when using the array form of `system()` to prevent shell injection, malicious inputs starting with `-` (like `--upload-pack`) can be interpreted as command-line options by the underlying tool (`git`), leading to potential remote code execution or arbitrary file writes.
**Prevention:** Always use the `--` separator before passing untrusted variable arguments to external commands (e.g., `cmd << "--" << url`) to ensure they are parsed strictly as positional parameters.
## 2026-07-18 - [Argument Injection in Git Ls-Remote Command]
**Vulnerability:** An argument injection vulnerability existed in `lib/rulepack/bump.rb` where user-controlled `url` and `ref` inputs were passed directly to `git ls-remote` via `Open3.capture3('git', 'ls-remote', url, ref)` without an end-of-options separator (`--`).
**Learning:** Similar to the `git clone` vulnerability, malicious inputs starting with `-` (like `--upload-pack`) could be interpreted as command-line options by `git ls-remote`, leading to potential remote code execution or unauthorized arbitrary operations.
**Prevention:** Always use the `--` separator before passing untrusted variable arguments to external commands (e.g., `Open3.capture3('git', 'ls-remote', '--', url, ref)`) to ensure they are parsed strictly as positional parameters.
